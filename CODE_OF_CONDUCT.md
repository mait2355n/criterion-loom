# Code of Conduct

## Purpose

Criterion Loom is a small research prototype about meaning-first audits for
Codex work. Discussion should stay useful, specific, and careful about public
claims.

## Audience And Use

Use this policy when participating in issues, pull requests, reviews, examples,
or documentation discussion in this repository.

## Expected Conduct

- Treat people with basic professional respect.
- Keep criticism tied to behavior, evidence, examples, or repository changes.
- Redact private prompts, customer data, secrets, and proprietary examples.
- Preserve the project boundary: this is not a security scanner, a release gate,
  a legal reviewer, or a replacement for human acceptance.
- Be explicit when a comment is fact, inference, hypothesis, or preference.

## Unacceptable Conduct

- Harassment, threats, slurs, or targeted personal attacks.
- Publishing private or sensitive material without consent.
- Repeated off-topic pressure after maintainers have declined a scope.
- Misrepresenting fixture results as broad natural-language accuracy.
- Treating LLM reviewer output as final human approval.

## Reporting

For ordinary conduct problems, open a GitHub issue with the smallest useful
public description. Do not include private material.

For sensitive reports, follow `SECURITY.md`. If private vulnerability reporting
is enabled, use it. Otherwise, open a minimal public issue that withholds
sensitive details and says that private details are available through an
appropriate channel.

Example public report shape:

```text
Summary: conduct concern in issue or pull request discussion
Location: link to public issue, pull request, or comment
Private material: none included
Requested action: maintainer review
```

## Enforcement

Maintainers may edit, hide, or close issues and pull requests that violate this
policy. Maintainers may also restrict participation when a pattern of behavior
is harmful to the repository's work.

Enforcement should be proportional, documented when practical, and focused on
protecting the repository's technical discussion.

## Scope Limits

This policy governs repository participation. It is not a promise of general
moderation outside this project, and it does not replace `SECURITY.md` for
sensitive reports.
